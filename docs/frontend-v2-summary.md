# 前端工作台 V2 阶段总结

日期：2026-07-10

## 阶段定位

V2 将原有静态页面升级为面向普通用户的 React 项目研究工作台。重点是让用户能够以自然语言描述需求，查看受证据约束的项目匹配结果，并在筛选、详情、推荐和对比之间连续操作。

本阶段不新增账号、服务端会话表或模型能力；检索、模型调用与质量闸门仍由后端负责。

## 已完成内容

### React 工作台与发布

- 新增 `frontend/` React + TypeScript 源码，构建产物发布到 `docs/app/`。
- 使用 Hash Router，提供项目匹配、项目筛选、研究推荐、项目详情与项目对比路由。
- 旧静态页面入口保留为跳转页，兼容原有查询参数。
- 顶部全局导航、API/发布环境标识、移动端抽屉和管理页入口在 React 路由中统一提供。

### 项目匹配对话

- `#/agent?api=1` 提供 GPT 式对话工作区：左侧本地会话历史、消息列表和底部输入区。
- 支持 SSE 草稿、取消生成、质量校验后的最终回答、拒答和规则降级状态。
- 候选集合与顺序只读取后端 `recommendations[]`；引用、检索片段、质量信息和原始上下文仅放入证据抽屉，不再由 citations 第一项猜测首选。
- 只有质量闸门通过且第一项 `eligibility=eligible` 时显示“当前归档内最匹配候选”；unknown、rejected 或质量失败时显示“暂无可确认首选”，并展示相对匹配分及满足/未满足要求。
- 会话仅保存在浏览器 `localStorage`，最多 10 个会话、每会话最多 20 轮；不保存密钥，不发送历史模型回答作为证据。
- 修复聊天布局：工作台采用全高网格，只有消息列表滚动，输入区始终位于工作区底部，不会被流式内容或项目卡挤出视口。

### 项目筛选与对比

- `/api/projects` 与 `/v1/projects` 新增非破坏性的 `offset`、`total`、`has_more` 分页元数据。
- 筛选页每页显示 50 个项目，筛选条件和页码同步到 URL，不再受默认 20 条结果限制。
- 新增本地项目对比暂存：`github_weekly_project_compare_v1`，最多 3 个仓库。
- 筛选页、推荐卡和对话项目卡均可加入对比；`repos` URL 参数优先于本地暂存，便于分享比较视图。
- 对比页提供空态选择器、添加/移除/清空操作，以及以语言、分类、来源、归档时间、Stars、增长、质量、风险和证据摘要为重点的矩阵。

## 当前边界与后续重点

项目匹配页已接入 P0-5 无状态追问 POST。浏览器只提交最近用户目标、候选仓库 ID、确认首选、模式和 resumable；不会提交历史 assistant answer、citations、evidence 或 prompt_context。无上下文的“继续/展开/嗯”直接澄清，不进入检索。

resume/refine 的候选限制已下推到 FTS5、vector、hybrid 内部检索；自然语言硬约束由确定性元数据和非模型增强 chunk 验证。澄清轮不展示项目卡或质量失败警告，无匹配轮展示冲突候选及原因。系统仍不建立服务端聊天会话。
