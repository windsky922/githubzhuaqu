# 项目路线图

## 第一阶段：最小可用版本

1. 根据近期搜索条件采集 GitHub 仓库。
2. 对仓库进行过滤和排序。
3. 生成 Kimi 周报或降级版基础周报。
4. 归档周报和运行摘要。
5. 在配置 Telegram 后发送消息。
6. 通过 GitHub Actions 每周自动运行。

## 第二阶段：增强数据质量

1. 在 `data/state/sent_repos.json` 中记录已推送仓库。已完成。
2. 抓取入选仓库的 README 内容。已完成。
3. 结合历史 Star 增量优化评分。已完成。

## 第三阶段：产品化输出

1. 生成 GitHub Pages 周报归档页面。已完成。
2. 增加趋势总结。已完成。
3. 在项目流程稳定后，再封装真正可用的 Codex 技能。已完成。

## 第四阶段：质量与可观测性增强

1. 增加入选原因记录，说明每个项目为什么进入周报。已完成。
2. 增加报告质量检查，减少格式错误、链接错误和模型误写。已完成基础版本。
3. 增加数据源分项统计，为多数据源扩展做准备。
4. 增强 GitHub Pages 首页摘要。
5. 增加项目自身安全检查，避免误提交密钥。已完成基础版本。
6. 增加入选仓库安全风险提示。已完成基础版本。

## 第五阶段：可扩展能力

1. 评估 GitHub Trending、GraphQL API 等补充数据源。
2. 评估邮件、飞书、企业微信等推送渠道。
3. 历史数据达到一定规模后，再评估 SQLite。
